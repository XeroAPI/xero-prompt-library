Build "The Executive" — a full-stack luxury single-product e-commerce site
selling one beltless trenchcoat at £2,495 (GBP).

DESIGN
Luxury aesthetic, restrained and editorial. No emojis anywhere.

Pages: landing (/), order form (/order), orders list (/orders),
order detail (/orders/:id), admin dashboard (/dashboard).

PRODUCT (server-side source of truth)
Define the product as a server-side constant: name, price 2495, currency GBP,
account code "200", available sizes and colours.
NEVER take unit price, currency, or account code from the client — only
size, colour, quantity, customer name, and email come from the request.

ORDER → XERO FLOW (on POST /orders)
Use the Xero connector to: upsert the contact by email, create an AUTHORISED
ACCREC invoice (14-day due, single line item describing the coat with
colour + size, chosen quantity, unit 2495, account code "200"), then fetch
the online invoice URL and direct a user to it in order to pay.

Persist every order to the database. If ANY Xero sub-call fails, save the order
with status "failed" plus the error message.
Follow the xero-integration-single-account skill for all connector mechanics, paths,
scopes, headers, and date/money normalisation.
